가구 판매 사이트 - Forest Of Furniture

사용된 기술들

Firebase의 Authentication과 Realtime Database이 적용된 웹사이트를 제작해 보았습니다. Firebase 인증을 사용하여 사용자가 구글로 로그인하거나 서버에 등록된 이메일과 비밀번호를 입력하여 사이트에 로그인할 수 있습니다. onAuthStateChanged 메서드를 사용해 변경된 인증을 관리하고, 로그인 사용자가 사이트를 다시 방문했을 때 세션에 저장된 인증을 가져오도록 구현하였습니다. 또한, 관리자 유저를 서버에 등록하여 admin 사용자를 구분할 수 있도록 구현하였습니다. 실시간 데이터베이스에서는 admin 사용자의 uid, 유저별 장바구니, 관리자가 등록한 상품 데이터를 실시간으로 공유하고 업데이트하여 수신하도록 동기화하였습니다.

Cloudinary를 사용하여 관리자 유저가 등록한 이미지를 손쉽게 업로드하고 url을 전달받아 데이터 베이스에 전달할 수 있도록 구현하는 연습을 하였습니다.

페이지와 컴포넌트, UI 컴포넌트를 분리하고, 컴포넌트와 로직을 분리하여 관리할 수 있도록 구현하였습니다. useContext로 최상위 컴포넌트에서 인증을 관리하고, firebase 인증, 실시간 데이터 관련 api와 cloudinary에 이미지 url을 받아오는 api를 나누어 목적별로 로직을 관리하기 쉽도록 구현해 보았습니다.

useQuery 라이브러리를 사용하여 불필요한 네트워크 통신을 방지하였습니다. staleTime을 1분으로 적용하여 1분간만 캐시 되도록 설정하였습니다. 캐시 된 경우 변경된 데이터를 적용시키기 위해서 useMutation를 설정하고 onSuccess인 경우 invalidateQueries가 적용될 수 있도록 로직을 구현하였습니다.

React-Router 라이브러리를 사용하여 기본 경로와 하위 경로들을 설정하였습니다. ProtectedRoute 컴포넌트를 생성하여 페이지 컴포넌트를 감싸고, ProtectedRoute에서 로그인 유무와 관리자 유저를 확인하여 인증이 되지 않은 유저는 최상위 경로로 이동시키는 경로 보호를 적용시켰습니다.

별도의 CSS 파일 없이 컴포넌트에서 스타일링이 가능한 Tailwind 라이브러리를 사용하였습니다. config 파일의 extend 속성을 적극 활용하여 필요한 스타일을 정의하여 적용시켜보았습니다.

적용된 기능들

  • 비회원인 경우 헤더에 로그인 버튼이 활성화되고 홈 화면과 상세페이지만 접근이 가능합니다. 비회원이 상품 상세페이지에서 장바구니에 추가하는 경우 로그인 안내 메시지가 출력됩니다.

  • 로그인 페이지에서 이메일 로그인과 구글 계정 로그인이 가능합니다. 비밀번호는 우측 버튼을 토글 하여 확인할 수 있고 로그인에 실패한 경우 안내 메시지가 출력됩니다.

  • 일반 회원인 경우 장바구니 기능을 사용할 수 있습니다. 상품 상세페이지에서 옵션을 선택하여 상품을 장바구니에 추가할 수 있습니다.

  • 관리자 유저인 경우 장바구니 기능을 테스트할 수 있고 제품 등록 기능을 사용할 수 있습니다. 해당 페이지에 등록한 제품은 실시간으로 반영되며, 내용 지우기 버튼을 클릭하면 입력한 내용이 모두 삭제됩니다.

  • 관리자 유저인 경우 제품 상세 페이지 상단에 제품 내용 수정 버튼이 활성화되고 클릭 시 제품 내용 수정 페이지로 이동됩니다. 해당 페이지는 경로 보호가 적용되어 비회원이나 일반 회원은 접근할 수 없습니다.

  • 제품 내용 수정 페이지로 이동 시 입력창에 제품의 기존 데이터가 기본으로 설정됩니다. 변경사항 없이 수정 버튼을 클릭하면 경고 메시지가 출력됩니다. 이미지 변경 없이 내용만 수정하면 기존 이미지가 적용됩니다.

  • 장바구니 페이지에서 -, + 버튼을 클릭하여 상품의 개수를 변경할 수 있습니다. 상품이 1개인 경우에 - 버튼을 클릭하면 최소 주문 수량이라는 안내 문구가 출력됩니다. 모든 제품의 금액과 배송비를 합산한 총액이 하단에 계산되어 출력됩니다.

  • 반응형 웹 디자인을 적용해 디바이스 크기에 따라 UI가 변경됩니다.
