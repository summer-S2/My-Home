Youtube clone app - Yuntube

사용된 기술들

YouTube API를 사용하여 YouTube와 상호작용하는 YouTube Clone App을 구현하였습니다. API Explorer를 사용하여 실시간 데이터에서 메서드를 호출하고 API를 요청하여 응답받은 JSON 데이터를 활용해 보는 연습을 할 수 있었습니다. YouTube Data API가 제공하는 하루 할당량을 효율적으로 관리하기 위해 Postman으로 받아온 JSON 파일을 저장하여 mock 데이터로 활용하였고, 실제 네트워크 통신과 mock 데이터 사용을 편하게 구분할 수 있도록 apiClient를 나누어 사용하였습니다.

프로젝트를 진행하면서 네트워크 통신을 하여 전달받은 데이터를 관리하고 화면에 구성하는 방법을 연습하였습니다. axios 라이브러리를 적용시켜 응답받은 JSON 데이터를 useContext로 관리하고, 모든 자식 컴포넌트에서 사용할 수 있도록 구현하였습니다. 또한 컴포넌트와 로직을 분리하여 컴포넌트에서는 UI에만 집중할 수 있고 로직은 api 폴더 내에서만 관리할 수 있도록 구현하였습니다.

React-Router 라이브러리를 프로젝트에 적용하여 클라이언트 사이드 라우팅이 가능하도록 구현하였습니다. 기본 경로에 중첩 경로를 구성하여 고정되는 컴포넌트와 URL에 따라 변경되는 컴포넌트인 Outlet을 설정하였습니다. 또한, 자식 경로에 Dynamic Segments 적용시켜 useParams()을 통해 데이터를 가져오는 동적 라우팅이 가능하도록 세분화하였습니다.

useContext를 사용하여 데이터를 요청할 때, useQuery 라이브러리를 적용시켜 불필요한 네트워크 통신을 방지할 수 있도록 구현하는 연습을 하였습니다. 요청받은 데이터를 1분만 캐싱 하여 신선한 데이터가 유지될 수 있도록 구현하였습니다. 또한 쿼리 키 배열에 키워드, id와 같은 변수를 할당하여 항목별로 데이터를 식별할 수 있도록 세분화하였습니다.

컴포넌트와 로직을 분리하였기 때문에 직관적으로 컴포넌트의 스타일을 관리할 수 있는 Tailwind 라이브러리를 사용하였습니다. className 속성에 JSX 문법을 활용하여 상태에 따라 다른 스타일이 적용되도록 구현하였고, tailwind.config 파일을 활용해 확장된 프로젝트만의 스타일을 적용시켜 보았습니다.

적용된 기능들

  • 화면 왼쪽 상단에 로고를 클릭하면 홈 화면 경로로 이동합니다.

  • 검색 바와 로고가 담긴 헤더는 화면에 고정하였고 검색 입력창에 키워드를 입력하면 키워드 검색이 가능합니다. 다른 페이지에서 뒤로 가기 버튼을 사용해 검색 페이지로 되돌아가는 경우 검색했던 키워드가 검색 입력창에 남아있습니다.

  • 동영상 카드를 클릭하면 상세 페이지로 이동합니다. 상세 페이지에서는 클릭한 동영상과 관련된 동영상이 화면에 추천됩니다.

  • 반응형 웹 디자인을 적용해 디바이스 크기에 따라 UI가 변경됩니다.
